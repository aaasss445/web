HTML 요소

HTML 요소는 일반적으로 시작 태그와 종료 태그로 구성되며 내용은 다음 사이에 삽입된다.

<tagname>내용이 여기에 있다 이거야...</tagname>

HTML 요소는 시작태그에서 종료 태그까지의 모든것이다.

<p>첫 번째 단락이다 이거다....</p>

시작 태그요소 내용끝 태그
<h1>My First Heading</h1>
<p>My first paragraph</p>
<br>

내용이 없는 HTML 요소를 빈 요소라고 한다고 하네.. 빈 요소에는 <br> 요소와 같은 태그가 있고.. 참고로 <br>는 줄 바꿈 이야....

중첩 된 HTML 요소

HTML요소는 중첩 될 수 있다.(요소안에 요소가 포함될수 있다...)

모든 HTML 문서는 중첩 된 HTML 요소로 구성된다.

이 예제에는 4가지 HTML 요소가 있다.

<!DOCTYPE html>
<html>
<body>

<h1>My First Heading</h1>
<p>My first paragraph</p>

</body>
</html>

예제 설명

<html>요소는 전체 문서를 정의

그것은 시작태그는 <HTML>, 종료태그는 </HTML>을 가지고 있다.

&lhtml>요소 안에는 <body>요소가 있다.

<html>
<body>

<h1>My First Heading</h1>
<p>My first paragraph</p>

</body>
</html>

<body>요소는 문서 본문을 정의 한다.

그것은 시작태그는 <body>, 종료태그는 </body>을 가지고 있다.

<body> 내부 요소는 두 개의 다른 HTML 요소 이다. <h1>및 <p>

<body>

<h1>My First Heading</h1>
<p>My first paragraph</p>

</body>

<h1>요소는 제목을 정의

그것은 시작태그는 <h1>, 종료태그는 </h1>을 가지고 있다.

요소 내용은 My First Heading 이다.

<h1>My First Heading</h1>

<p>요소는 단락을 정의

그것은 시작태그는 <p>, 종료태그는 </p>을 가지고 있다.

요소 내용은 My first paragraph 이다.

<p>My first paragraph</p>

종료 태그를 잊지 마라

종료 태그를 잊어 버린 경우에도 HTML 요소가 올바르게 표시된다.

<html>
<body>

<p>This is a paragraph
<p>This is a paragraph

</body>
</html>

닫는 태그는 선택 사항으로 간주되므로 위의 예는 모든 브라우저에서 작동한다.

하지만 이것에 의존하면 안된다. 종료 태그를 잊어 버린 경우 예기치 않은 결과 및 모류가 발생할 수 있다.

빈 HTML 요소

내용이 없는 HTML 요소를 빈 요소라고 한다.

<br>는 닫는 태그가 없는 빈 요소이다.(<br>태그는 줄 바꿈을 정의 함)

<p>This is a <bt> parapraph with a line break.</p>

빈 태그는 <br/>와 같이 여는 태그에서 "닫힐"수 있다.

HTML5는 빈 요소를 닫을 필요가 없다. 그러나 보다 엄격한 유효성 검사를 원하거나 XML 파서가 문서를 읽을 수 있게 하려면 모든 HTML 요소를 올바르게 닫아야 한다.

대소 문자를 구분하지 않는 HTML

HTML 태그는 대소 문자를 구분하지 않는다. <P>는 <p>와 동일

HTML5표준에는 소문자 태그가 필요하지 않지만 W3C 는 HTML에서 소문자를 권장 하며 XHTML과 같은 보다 엄격한 문서 유형에서는 소문자를 요구
